git笔记

发表于 2021-05-07 更新于 2021-05-14 分类于 版本控制 阅读次数： Valine：

git命令行操作

初始化git

git init

设置签名

项目级别/仓库级别：仅在当前本地库范围内有效

git config user.name hui

git config user.email 1764501567@qq.com

信息保存位置：./.git/config 文件

系统用户级别：登录当前操作系统的用户范围

git config –global user.name hui

git config –global 1764501567@qq.com

信息保存位置：~/.gitconfig 文件

注: 就近原则,项目级别优先于系统用户级别

其他常规操作

状态查看   git status

添加  git add[file name] 

提交 git commit -m’提交日志’ [file name] 

查看历史提交记录

git log

git reflog ( 最常用 )

git log –pretty=oneline 

git log –oneline 

多屏显示控制方式： 空格向下翻页 , b 向上翻页 , q 退出

前进后退版本

git reset –hard [局部索引值]

git reset –hard HEAD^

注：一个^表示后退一步，n 个表示后退 n 步

git reset –hard HEAD~n

注：表示后退 n 步

reset 命令的三个参数对比

–soft

​ 仅仅在本地库移动 HEAD 指针

–mixed

​ 在本地库移动 HEAD 指针

​ 重置暂存区

–hard

​ 在本地库移动 HEAD 指针

​ 重置暂存区

​ 重置工作区

比较文件差异

git diff [文件名]

将工作区中的文件和暂存区进行比较

git diff [本地库中历史版本] [文件名]

将工作区中的文件和本地库历史记录比较 ，如果不带文件名则比较多个文件

分支管理

查看分支 git branch -v

创建分支 git branch [分支名]

切换分支 git checkout [分支名]

合并分支 git merge [有新内容分支名]

解决合并分支的冲突

​ 第一步：编辑文件，删除特殊字符

​ 第二步：把文件修改到满意的程度，保存退出

​ 第三步：git add [文件名]

​ 第四步：git commit -m “日志信息”

​ 注意：此时 commit 一定不能带具体文件名

github基本命令

git clone [远程地址] 克隆项目

git remote -v 查看当前所有远程地址别名

git remote add [别名] [远程地址] 添加远程地址

git push [别名] [分支名] 推送

拉取

pull=fetch+merge

git fetch [远程库地址别名] [远程分支名]

git merge [远程库地址别名] [远程分支名]

git pull [远程库地址别名] [远程分支名]

冲突解决

如果不是基于GitHub远程库的最新版本所做的修改，不能推送，必须先拉取

拉取下来后如果进入冲突状态，则按照“分支冲突解决”操作解决即可
